# Краткий отчёт для научного руководителя: визуальная релокализация и сравнение с оптическим трекингом

## Цель работы на этом этапе

По видео сцены построить датасет (COLMAP), обучить нейросеть (PoseNet+) предсказывать 6DoF-позу камеры по одному кадру, на тестовом видео получить предсказания и сравнить траекторию с **эталоном оптического трекинга** (`AL_Optic.csv`) после геометрического выравнивания систем координат.

## Что реализовано в репозитории

1. **Пайплайн v2** (`scripts/run_pipeline_v2.sh`): COLMAP по train → обучение на `train_v2_scene` → COLMAP по test → инференс → сравнение с оптикой (переменные `TRAIN_VIDEO`, `TEST_VIDEO`, `OPTIC_CSV`, `DOWNSAMPLE_FPS`).

2. **Конфиг обучения** `configs/train_v2.yaml` (PoseNet+, ResNet-34, сцена `train_v2_scene`).

3. **Сравнение ML с оптикой** `scripts/compare_ml_optic.py`:
   - выравнивание позиций **Umeyama** (масштаб + поворот + перенос): \(p_\mathrm{opt} \approx s R p_\mathrm{colmap} + t\);
   - учёт жёсткого крепления **датчик–камера** (пресет по схеме осей или своя матрица `--sensor_to_camera_rowmajor`);
   - **четыре базовые метрики**, как обсуждалось: макс./среднее отклонение по позиции (м); макс./среднее по сферическому расстоянию ориентации на SO(3) (°);
   - дополнительно: медиана, RMSE по позиции; **относительная** ошибка ориентации между соседними кадрами; опционально **Kabsch** по всей траектории для оценки постоянного поворота между экспортом оптики и цепочкой COLMAP (`--kabsch_orientation_calibration`).

4. **Подготовка данных под обрезанное видео и очистку оптики**:
   - `scripts/trim_optic_csv.py` — обрезка `AL_Optic.csv` по длительности видео (`ffprobe`) или по `t_min`/`t_max`;
   - `scripts/optic_angular_peaks.py` — анализ пиков |ω| по `ang_vel_*`, фильтрация CSV, JSON интервалов для согласованной нарезки видео.

5. **COLMAP**: `scripts/make_dataset.sh` выбирает **крупнейшую** успешную реконструкцию в `sparse/` (`scripts/pick_largest_sparse_model.py`).

Датасеты `train_v2_scene/`, `test_v2_scene/`, видео `*.mp4`, чекпоинты и `outputs/` в репозиторий **не входят** (см. `.gitignore`). Файлы `AL_Optic*.csv` также игнорируются как локальная телеметрия; для воспроизведения их нужно положить в корень проекта вручную.

## Численные результаты (ориентиры; точные значения — в логах/`outputs/*.json` при прогоне)

### Обучение

- Модель: **PoseNet+** по конфигу `configs/train_v2.yaml`.
- На использованной машине обучение шло на CPU; срабатывала **ранняя остановка** (порядка 40-го эпохи), чекпоинт `outputs/posenet/best.pth` (не в git).

### Сравнение с оптикой

**Полный тестовый ролик** (типичный прогон: ~2350 кадров, FPS из метаданных видео ≈ 30):

| Показатель | Порядок величины |
|------------|------------------|
| Позиция: среднее отклонение | ~**0,04 м** |
| Позиция: максимум | ~**0,27 м** |
| Ориентация (абсолютная), среднее | ~**120–130°** |
| После оценки **R_kabsch** (постоянный поворот по траектории): среднее / медиана | ~**60° / ~35°** |
| Относительная ориентация (соседние кадры), среднее / медиана | ~**4–5° / ~2°** |

**Обрезанное тестовое видео** `test_v2_cuted.mp4` (~33 с, оптика `AL_Optic_cuted.csv`, FPS ≈ **29,64** из `ffprobe`, ~988 кадров после пересборки `test_v2_scene`):

| Показатель | Порядок величины |
|------------|------------------|
| Позиция: среднее / максимум | ~**0,06 м** / ~**0,23 м** |
| Ориентация (абсолютная), среднее | ~**155°** |
| После **R_kabsch**: среднее / медиана | ~**76° / ~62°** |
| Относительная ориентация, среднее / медиана | ~**6,5° / ~3,8°** |

**Интерпретация:** позиция после Umeyama согласуется с оптикой на приемлемом уровне для комнатной сцены. Большие **абсолютные** углы в основном связаны с несовпадением объявленных осей экспорта оптики и системы COLMAP/камеры (и крепления датчик–камера); **относительно** шаги поворота ML и оптики ближе, что указывает на систематический сдвиг рамок. **R_kabsch** — осмысленная **вспомогательная** оценка этого сдвига по данным, но **не заменяет** калибровку по документации оптики и измеренное преобразование датчик↔камера.

## Чего не хватает для «физически выверенной» абсолютной ориентации

1. Формальное описание кватерниона в `AL_Optic.csv` (порядок компонент, world→тело / тело→world).
2. Измеренная (или по CAD) матрица и рычаг **датчик ↔ камера** вместо пресета по схеме.
3. При необходимости — уточнение **синхронизации** записи видео и оптики (`--time_offset_sec` в скрипте сравнения).

## Команды для воспроизведения (после размещения данных локально)

```bash
# Полный пайплайн (видео и CSV в корне проекта)
./scripts/run_pipeline_v2.sh

# Только сравнение (есть pred_poses.csv и обрезанная оптика)
python scripts/trim_optic_csv.py -i AL_Optic.csv -o AL_Optic_cuted.csv --match_video test_v2_cuted.mp4
python scripts/compare_ml_optic.py \
  --optic_csv AL_Optic_cuted.csv \
  --pred_csv test_v2_scene/pred_poses.csv \
  --frames_fps <из ffprobe avg_frame_rate> \
  --kabsch_orientation_calibration \
  --export_json outputs/metrics_ml_vs_optic_cuted.json
```

Подробности — в **README.md**, раздел «Пайплайн train_v2 / test_v2 и сравнение с оптическим трекингом».
